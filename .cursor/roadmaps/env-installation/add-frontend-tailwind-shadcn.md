## Додавання нового Frontend у Turborepo (Tailwind + shadcn)

Цей документ описує, як **після першого встановлення** (коли Tailwind і shadcn вже налаштовані “спільно” в monorepo) додати **новий Next.js frontend** і забезпечити, щоб він коректно використовував:
- централізований Tailwind (за підходом “with-tailwind”)
- shadcn/ui, згенерований у спільному пакеті

> Контрольна точка: перший раз Tailwind/shadcn треба зробити один раз. Далі — лише підключення для нових фронтендів.

Спираємося на ідею з прикладу Turborepo: [with-tailwind](https://github.com/vercel/turborepo/tree/main/examples/with-tailwind).

---

## 0. Передумови (що має існувати після початкового setup)

1. У monorepo вже встановлені й підключені залежності для Tailwind/PostCSS (централізовано).
2. Налаштування Tailwind зроблено так, щоб Tailwind “бачив” використання класів у спільному UI пакеті (наприклад, `packages/ui`), а не лише в одному додатку.
3. shadcn/ui компоненти згенеровані у спільний пакет (типово `packages/ui/...`) і цей шлях відомий команді.
4. Для кожного Next.js frontend увімкнено:
   - `transpilePackages`, щоб Next коректно збирав код спільного пакета з UI.
5. У кожному frontend є `app/globals.css` (або еквівалент), де підключені Tailwind директиви `@tailwind ...` (або еквівалентний механізм імпорту).

Якщо хоча б один пункт не виконано — повернись до першого setup і доведи його до стану “working”.

---

## 1. Створення нового frontend app

1. Створи новий Next.js застосунок у `apps/<new-app>` (будь-яким зручним способом, який ти використовуєш для Turborepo).
2. Додай його до `turbo.json` (або переконайся, що існуюча конфігурація автоматично підхоплює нові apps).
3. Переконайся, що у `apps/<new-app>/package.json`:
   - є `@repo/ui` як `workspace:*`
   - scripts співпадають за назвою з рештою apps (щоб `turbo run dev/build/lint` працювали однаково)

---

## 2. Підключення Tailwind у новому app

Мета: **Tailwind для нового app має сканувати класи з твого спільного UI пакета**.

1. Переконайся, що у `apps/<new-app>` є:
   - `tailwind.config.ts/js` (або конфіг, який наслідується з централізованого)
   - `postcss.config.ts/js` (якщо в твоєму setup він очікується на рівні app)
2. Налаштуй `tailwind.config.*` так, щоб `content` включав:
   - файли самого app (`apps/<new-app>/**`)
   - спільний UI пакет (`packages/ui/**`) та/або шлях до shadcn компонентів
3. У `apps/<new-app>/app/globals.css`:
   - додай Tailwind директиви `@tailwind base;`, `@tailwind components;`, `@tailwind utilities;`
   - збережи ваші theme/CSS-перемінні, якщо вони визначені в попередніх apps (наприклад, `--background`, `--foreground`)

> Якщо в монорепо використовується спільна конфігурація Tailwind (через пакет або імпорт), то у новому app має бути мінімальна “обгортка”, а не копія всієї конфігурації.

---

## 3. Підключення shadcn/ui в новому app

Мета: **не генерувати shadcn заново в кожному app**, а використовувати спільні компоненти.

1. У `apps/<new-app>` імпорти роби з спільного пакета, який містить zгенеровані shadcn компоненти (за тим, як у вас організований `packages/ui`).
2. Переконайся, що в `apps/<new-app>/next.config.js` присутній `transpilePackages` для вашого UI пакета:
   - типово: `transpilePackages: ["@repo/ui"]`
3. Перевір, що в `packages/ui/package.json` експорти (exports) дозволяють доступ до тих компонентів, які ти імпортуєш у frontend.

Якщо у вас ще не зроблено “правильні exports” для глибокого шляху компонентів (наприклад, коли shadcn згенерував в `src/components/ui/...`), це потрібно вирішити на рівні `packages/ui`, а не копіювати генерацію в app.

---

## 4. Швидка перевірка після налаштування

1. Запусти:
   - `pnpm run dev --filter=<new-app>` (або `turbo dev --filter=<new-app>`)
2. Відкрий сторінку, де рендериться shadcn-компонент, наприклад `Button`.
3. Перевір:
   - стилі застосовуються (не “unstyled”)
   - немає помилок імпорту з `@repo/ui`

Якщо стилі не застосовуються — найчастіші причини:
- у `tailwind.config.*` у `content` не включено `packages/ui/**`
- `transpilePackages` відсутній (або не той пакет)
- shadcn компоненти імпортуються з шляху, який не експортовано в `packages/ui`

---

## 5. Що робити, коли ви додаєте шіфти/зміни в UI

1. Зміни shadcn (нові/оновлені компоненти) роби у спільному місці (у пакеті, де згенеровано shadcn).
2. Після зміни UI:
- нові apps мають автоматично підхоплювати стилі, якщо `tailwind.config` правильно включає `packages/ui/**`
- якщо змінено структуру компонентів — перевір `packages/ui/package.json` exports

---

## 6. Чекліст “done”

- [ ] `apps/<new-app>` має `@repo/ui`
- [ ] `tailwind.config.*` у `apps/<new-app>` включає `packages/ui/**` в `content`
- [ ] у `app/globals.css` є Tailwind директиви (або аналог)
- [ ] `next.config.js` містить `transpilePackages: ["@repo/ui"]`
- [ ] імпорт shadcn компонентів з `@repo/ui/...` працює
